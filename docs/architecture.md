This is LLD (low level design) learning react application.
../content directory will contain all required LLD markdown files for which we need to render react pages.
We will have LLD questions from domains:
1. Core Infrastructure & Storage
2. Concurrency & Scheduling
3. Networking & Protocols
4. Messaging & Event Systems
5. API & Access Control
6. File Systems & IO
7. Payments & Transactions
8. Observability & Reliability
9. Caching & Proxies
10. Practical System components
etc etc.

What these topics will multiple questions inside them, and each question solution should have:
1. core classes/interfaces
2. database schemas
3. data structures with justification
4. concurrency handle
5. API contracts
6. Tradeoffs
7. Happy paths
etc etc.

Above is not strict, but somewhat we will try to follow this structure for each question.
