## Study Strategy

**Pattern mastery order:**
1. BFS / DFS on grids → Connected components → Topo sort
2. Dijkstra → Bellman-Ford → Floyd-Warshall
3. Union-Find → MST (Kruskal, Prim)
4. Cycle detection → Bipartite → Graph coloring
5. SCC / Bridges / Articulation points
6. Euler paths → Hamiltonian / TSP (bitmask)
7. Implicit / state-space graphs → 0-1 BFS → Bidirectional BFS
8. LCA / Binary lifting → Functional graphs
9. Graph + DP → Graph + Greedy
10. Flow / Matching (conceptual — rarely coded in interviews)

**Senior/Staff round priorities:**
- **Dijkstra with modified state**: (787) K-stop shortest path, (1293) obstacle elimination, (1928) time-constrained paths — these test state augmentation skills
- **Union-Find mastery**: path compression + union by rank is baseline; test with (684/685) redundant connections, (1697) offline sorted queries, (1579) edge removal
- **Topological sort + DP on DAGs**: (2050) Parallel Courses III, (329) longest increasing path, (269) alien dictionary
- **Multi-source BFS**: (994) rotting oranges, (286) walls and gates, (317) shortest distance from all buildings
- **SCC / Bridges**: (1192) critical connections — Tarjan's is a must-know
- **Implicit graph modeling**: recognizing when a problem IS a graph problem — (752) lock combinations, (773) sliding puzzle, (913) Cat and Mouse game states
- **Functional graphs**: (2127) maximum employees invited, (2360) longest cycle — successor graph + cycle detection combo

**Common interview traps:**
- Forgetting visited set → infinite loop on undirected graphs
- Using DFS for shortest path (BFS is correct for unweighted)
- Not handling disconnected components
- Mixing 0-indexed and 1-indexed nodes
- Not considering edge cases: self-loops, parallel edges, empty graph

**Total: ~300+ problems across 30 categories**
