## Study Strategy

**Pattern mastery order:**

1. Basic Kth element / Top-K → Min/Max heap choice
2. Two heaps (median stream) → K-way merge
3. Greedy + heap (scheduling, regret-based exchange)
4. Sort-then-sweep with heap (meeting rooms family)
5. Dijkstra as heap-driven BFS → modified Dijkstra
6. Lazy deletion pattern → sliding window + heap
7. Reorganize / rearrange (cooldown-based greedy)
8. Heap + prefix/suffix → multi-key comparator heaps
9. Design / online / stream processing problems
10. Trapping Rain Water II / Skyline (complex multi-pattern)

**Senior/Staff round priorities:**

- **Two heaps (median)**: (295) Find Median from Data Stream is an absolute must — know the invariant: max-heap for lower half, min-heap for upper half, balance sizes. (480) Sliding Window Median extends it with lazy deletion
- **Regret-based greedy**: the pattern where you greedily pick, then "undo" via max-heap — (630) Course Schedule III, (871) Refueling Stops, (1642) Furthest Building. Interviewers love this because it tests whether you see that greedy alone fails but greedy + heap works
- **Sort-then-sweep**: sort by one dimension, heap on another — (857) Hire K Workers (sort by ratio, min-heap on quality), (1383) Maximum Performance (sort by efficiency desc, min-heap on speed), (2542) Maximum Subsequence Score. This is a Staff-level pattern
- **K-way merge**: (23) Merge k Sorted Lists, (632) Smallest Range — the key insight is maintaining one element per list in the heap
- **Dijkstra variants**: every modified Dijkstra problem (augmented state, multiple constraints) uses the heap as its core engine
- **Lazy deletion**: know when and why — when removals are infrequent relative to inserts, mark-and-skip is cheaper than a balanced BST. (480) and (2034) are canonical
- **The Skyline Problem**: tests sweep line + max-heap + lazy deletion + edge case handling — a top-tier interview problem

**Key implementation decisions:**

- **Min-heap vs Max-heap**: in C++ `priority_queue` is max-heap by default; negate values or use `greater<>` for min-heap. In Python, `heapq` is min-heap; negate for max
- **Heap vs Balanced BST**: heap gives O(1) peek + O(log n) insert/remove-top. If you need arbitrary removal or both min and max, use `multiset` (C++) or `SortedList` (Python)
- **Heap vs Quickselect**: for one-shot Kth element, Quickselect is O(n) average. Heap is better for streaming / online Kth
- **Heap size K vs N**: maintaining a heap of size K gives O(n log K); useful when K << N

**Common interview traps:**

- Forgetting that C++ `priority_queue` is max-heap (not min)
- Using heap when a monotonic deque gives O(n) — sliding window max
- Not handling ties in custom comparator (undefined behavior in some languages)
- Lazy deletion without checking if heap top is stale before using it
- Forgetting to handle empty heap edge cases
- Using heap for Kth pair distance (TLE) when BS + two-pointer is O(n log n)

**Total: ~280+ problems across 25 categories**
