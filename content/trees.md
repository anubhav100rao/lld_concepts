## Study Strategy

**Pattern mastery order:**

1. DFS traversals (pre/in/post, iterative + recursive) → BFS level order
2. Tree construction / serialization (105, 106, 297)
3. BST operations (validate, insert, delete, kth, recover)
4. LCA family (236, 235, 1644, 1650, 1483)
5. Path problems (112, 437, 124, 543)
6. Tree DP (337, 968, 979, 1339)
7. Vertical / boundary / diagonal traversal
8. Tree symmetry / comparison / isomorphism
9. Tree modification / transformation
10. Rerooting DP (834, 2846)
11. Segment tree / BIT
12. Binary lifting / Euler tour / HLD / centroid decomposition

**Senior/Staff round priorities:**

- **Binary Tree Maximum Path Sum (124)**: the gold standard tree DP problem. At each node, compute the best "straight path" (node + left or right child), propagate that up, while tracking the global best "bent path" (left + node + right). The key is returning the straight path but updating answer with the bent path
- **Serialize/Deserialize (297)**: tests design thinking — preorder with null markers, level-order with null markers, or parenthesized representation. Know the tradeoffs. Interviewers test edge cases: empty tree, single node, skewed tree
- **Path Sum III (437)**: prefix sum on tree paths — pass a hashmap of prefix sums down the DFS, add current sum, look up (current - target). This is the tree version of "Subarray Sum Equals K" (560). Staff-level insight
- **Binary Tree Cameras (968)**: greedy on tree with 3 states (uncovered, covered, has camera). Post-order DFS, decide bottom-up. The greedy insight: never place cameras on leaves, always on their parents
- **Sum of Distances in Tree (834)**: the canonical rerooting DP. First DFS: compute subtree sizes and root's answer. Second DFS: re-root — when moving root from parent to child, nodes in child's subtree get 1 closer, rest get 1 farther. `ans[child] = ans[parent] - subtree[child] + (n - subtree[child])`
- **LCA with binary lifting (1483)**: preprocess ancestors at powers of 2 for each node. Query in O(log n). Extends to path max/min queries. Know the technique: `up[v][k] = up[up[v][k-1]][k-1]`
- **BST Iterator (173)**: controlled iterative inorder — maintain a stack, push all left children, pop to get next, then push right child's lefts. Tests understanding of what the call stack does in recursive inorder
- **Morris Traversal**: O(1) space inorder by temporarily threading right pointers. Phase 1: find predecessor, set thread. Phase 2: follow thread back, remove it. Rarely asked to code but understanding it deeply impresses

**Key patterns to internalize:**

```
Pass-down vs Collect-up:
  - Pass-down: carry state FROM root (max so far, prefix sum, depth)
  - Collect-up: compute answer FROM leaves (subtree sum, height, diameter)
  - Many problems need BOTH (124: collect max path up, but also compute global bent path)

DFS return value design:
  - Return ONE thing that the parent needs
  - Track the GLOBAL answer as a separate variable
  - Example (124): return max single-arm path, track global max bent path

Rerooting formula:
  - ans[child] = f(ans[parent], subtree[child], n)
  - Usually involves adding/subtracting subtree contributions
  - Must precompute subtree aggregates in first DFS

BST inorder = sorted order:
  - Any "kth smallest", "range sum", "validate" problem → think inorder
  - Reverse inorder = largest to smallest (for Greater Tree)
```

**Common interview traps:**

- **Confusing node value with return value**: in tree DP, what you return to parent is often different from the answer you compute locally
- **Null handling**: forgetting base case for null nodes — return 0, -inf, or empty depending on context
- **Off-by-one in depth**: is root depth 0 or 1? Clarify with interviewer
- **BST validation**: passing (min, max) range down is correct. Doing `left.val < node.val < right.val` is WRONG (doesn't check grandchildren)
- **LCA with parent pointer**: the "two linked lists intersection" trick — walk both pointers, on reaching null switch to other root. They meet at LCA
- **Serialize/Deserialize**: using level-order is simpler but wastes space. Preorder + null markers is compact. Don't forget the delimiter between values
- **Maximum Width**: use index math (left = 2*i+1, right = 2*i+2) not actual counts. Offset indices per level to prevent overflow
- **Finding diameter**: the answer isn't necessarily through the root — must track global max across all nodes

**Total: ~400+ problems across 30 categories**
