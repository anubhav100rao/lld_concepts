## Study Strategy

**Pattern mastery order:**
1. Two pointers (opposite + same direction) → Sliding window (fixed + variable)
2. Prefix sum → Difference array / sweep line
3. Hash map on arrays → Subarray counting / contribution
4. Kadane's variants → In-place manipulation / cyclic sort
5. Sorting (Dutch flag, custom comparator, bucket/counting/radix)
6. Merge / D&C on arrays → Boyer-Moore voting
7. Intervals (merge, insert, scheduling)
8. Prefix/suffix precomputation → Monotonic trend detection
9. Bit manipulation on arrays → Array math / number theory
10. Design / online / randomization

**Senior/Staff round priorities:**

- **Sliding window mastery**: the variable-size sliding window template is tested everywhere. Know the "atMost(k) − atMost(k−1)" trick for "exactly K" problems (992, 1248, 930). The invariant: expand right, shrink left while condition holds, count valid windows
- **Prefix sum + hash map combo**: (560) Subarray Sum Equals K is the canonical pattern — prefix[i] − prefix[j] = k ⟹ look up prefix[j] = prefix[i] − k in a hashmap. Extends to modular arithmetic (523, 974), XOR (1310), and 2D (304)
- **Difference array**: the O(n) trick for range updates — add at start, subtract at end+1, prefix-sum to reconstruct. (1094) Car Pooling, (1109) Flight Bookings, (370) Range Addition
- **First Missing Positive (41)**: the ultimate in-place array problem — use the array itself as a hash table by placing each number at its correct index. Tests O(1) space thinking under pressure
- **Contribution technique**: for "sum of f over all subarrays", compute each element's contribution independently. Combine with monotonic stack for min/max variants (907, 2104, 2281). This is a staff-level pattern
- **Meet-in-the-middle (1755)**: split array in half, enumerate all subset sums for each half (2^20 each), sort + two-pointer or binary search to find closest to target. Rare but impressive
- **Merge sort-based counting**: (315, 493, 327) — count inversions / special pairs during merge sort by exploiting sorted halves. Alternative: BIT with coordinate compression

**Key sliding window template:**
```python
# Variable size — find longest/shortest window with condition
def sliding_window(arr, ...):
    left = 0
    result = 0
    state = ...  # window state (hashmap, count, sum, etc.)
    for right in range(len(arr)):
        # expand: add arr[right] to state
        while condition_violated(state):
            # shrink: remove arr[left] from state
            left += 1
        result = max(result, right - left + 1)  # or min, or count
    return result

# "Exactly K" via atMost trick:
def exactlyK(arr, k):
    return atMostK(arr, k) - atMostK(arr, k - 1)
```

**Key prefix sum template:**
```python
# Subarray sum equals K
def subarray_sum(nums, k):
    prefix = {0: 1}
    curr_sum = 0
    count = 0
    for num in nums:
        curr_sum += num
        count += prefix.get(curr_sum - k, 0)
        prefix[curr_sum] = prefix.get(curr_sum, 0) + 1
    return count
```

**Common interview traps:**
- Sliding window: shrinking from the wrong side, or forgetting to update state when shrinking
- Prefix sum: off-by-one in range queries — prefix[r+1] − prefix[l], not prefix[r] − prefix[l]
- In-place: modifying the array while iterating forward (use backward iteration or two-pass)
- Sorting stability: when multiple sort criteria matter, unstable sorts can break ties wrong
- Overflow: summing large arrays — use long / int64, not int
- Empty array / single element edge cases
- "Exactly K" problems: don't try direct — use atMost(K) − atMost(K−1)
- Dutch National Flag: confusing the three-pointer boundaries (lo, mid, hi)
- Difference array: forgetting the +1 offset on the end index
